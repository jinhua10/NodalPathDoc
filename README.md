## 关于开源与收费

**NodalPath 软件不开源。** 软件本身是免费的，暂时不向用户收取任何额外费用。

## 费用说明

所有费用由用户直接与 AI 服务商产生关系，例如你往 DeepSeek 充值、消耗 token 所产生的费用，我们不做任何中间加价。

**我们不做数据存储。** NodalPath 软件本身不会存储你的数据，因此你的数据安全由你自己掌控。

## 网站入口

我们会提供一个网站入口：[yumbo.top](https://yumbo.top)，用于承载 NodalPath 的 Web 端入口。

## 运营成本

由于软件本身不存储数据，运营成本仅在于网站 JS 服务器的托管与带宽。吞吐量完全由各 API 厂商负责承担，我们的服务端压力极小。

## 未来的可能性

或许以后我们会考虑从中收取费用，但方式不是向用户加价，而是与 AI 厂商合作。例如当用户量达到一定量级后，我们与 DeepSeek 等厂商洽谈合作，厂商给我们 token 优惠，用户在 NodalPath 中消耗 token 的费用不会变多，我们从中获取优惠差价。当然，这需要用户量达到有议价资格的量级。

但不必为我们的存活担忧——在设计初期就已想好，运营成本只在网站 JS 服务器的运营，这是一个极轻量的成本结构。

## 我为什么做 NodalPath

本质上，我想做一些产品去卖。但基于我对目前 AI 工具的使用和理解，我看到了一个问题：**AI 工具帮我们完成产品的 MVP 之后，存在大量的维护与功能迭代困境。**

### 对商业模式的思考

我持续思考这个世界的运转机制。举个例子：如果我想利用 AI 做一款游戏在 Steam 发布，通过分成获取收益——这条路我们都知道可以赚到钱，但真的去做时，会发现存在很多问题。

> 大型 3A 游戏显然不是一个人能完成的。即使你用上最先进的 AI 工具，这里面依旧需要投入大量成本，而且很可能存在很多问题。对于个人开发者来说，做一款 3A 大作是一件极其困难的事。

我思考了更深层的问题：**3A 大作是怎么产生的？个人开发与现有的 3A 大作相比缺乏了什么东西？这些东西能否在我能力范围内运用上？**

我得到的答案是：让一款游戏爆火的，从来不是单纯的产品技术，而是一个**顶级想法**，加上凑齐了资源，然后逐步完善。

### 比产品技术更重要的是想法和商业模式

这个道理是通用的。比产品技术更重要的是你的想法和商业模式。如果你本身就没有想通你做的产品商业模式是怎样的，那么你的产品就注定存在问题。

一款大型游戏里有很多功能模块，这些功能模块的确可以通过 AI 工具完成（当然也要考虑 AI 本身的局限）。**但在完成这个目标前，要推演出这个已存在的、爆火产品的底层原理。**

这就是我想做一款 AI 推理工具的原因。AI 的结果存在一些问题，需要人与它共同协作，才能完成一份完整的商业推演。

## "道" + "术" 理论

我认为个人开发者如果想要使用 AI 完成商业模式的构建，需要一个 **"道" + "术"** 的理论框架：

- **道**：规律，本身就存在的规律。是事物运行的底层逻辑。
- **术**：推理，是运用"道"的体现。是对规律的具体运用和实践。

我看待世界已发生的事件，就是以 **"道 + 术"** 的视角来看待的。NodalPath 正是这一理念的产物——帮助你去发现商业中的"道"，并用"术"去推演和实践。
把大模型比作计算器，而NodalPath调用大模型帮助用户建立自己的“道”，这种运用大模型本身的“道”去推演我们的“道”是`道 + 术`体现。
